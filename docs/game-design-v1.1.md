## 一、游戏整体状态机

### 1.1 房间生命周期

```
[创建房间] → LOBBY（等待玩家）
     │
     ↓（房主点击开始 & 满足人数条件）
PREPARATION（武将选择阶段）
     │
     ↓（所有玩家确认选将 或 60秒倒计时归零）
PLAYING（游戏进行中）
     │
     ↓（触发胜负判定）
SETTLEMENT（结算页面）
     │
     ↓（所有玩家确认 或 30秒后自动）
DISSOLVED（房间解散）
```

**状态说明**：
- `LOBBY`：可加入/退出，房主可修改房间配置（人数上限、游戏模式）
- `PREPARATION`：不可加入，可选武将，60 秒超时者随机分配默认武将
- `PLAYING`：核心游戏状态，见 §1.2
- `SETTLEMENT`：展示排名、成就、资产变化，按钮「再来一局」「返回大厅」
- `DISSOLVED`：所有连接断开，服务端清理房间数据

### 1.2 回合状态机（PLAYING 内部）

每一个玩家回合严格按以下状态顺序流转，不可跳过：

```
TURN_START
    │ 结算持续性状态（乐不思蜀、结盟倒计时、美人计冷却）
    │ 触发「回合开始」被动技能（荀彧：+100金）
    ↓
LOCKED（入狱状态特判）
    ├─ [已入狱] → JAIL_ACTION（只能：出狱操作 或 等待）
    │         └─ 出狱成功 → ROLL_DICE
    │         └─ 继续蹲狱 → TURN_END
    └─ [未入狱] → ROLL_DICE
         │
         ↓
    ROLL_DICE（掷骰子）
         │ 等待玩家点击「掷骰」按钮（或 15 秒超时自动掷）
         │ 触发赵云重掷选择弹窗（如持有）
         ↓
    MOVING（动画移动中）
         │ 棋子逐格动画移动
         │ 途中经过起点：自动结算起点奖励，播放经过动画
         ↓
    LANDING（落地结算）
         │ 根据格子类型进入对应子流程（见 §2）
         ↓
    CARD_ACTION（可选：使用锦囊卡）
         │ 若玩家手持锦囊卡，高亮「使用卡牌」按钮，等待 20 秒
         │ 若无卡牌或超时，跳过
         ↓
    UPGRADE_ACTION（可选：升级城池）
         │ 弹出「是否升级城池」面板，等待 15 秒
         │ 可升级多座城池，每次确认消耗费用
         ↓
    TURN_END
         │ 触发「回合结束」被动技能（孙权：换牌）
         │ 手牌上限检查（>5 张弹出弃牌面板，强制弃牌后继续）
         │ 破产检查（见 §5）
         │ 胜负检查（只剩 1 人 → SETTLEMENT）
         ↓
    [转移至下一位玩家] → TURN_START
```

### 1.3 超时自动操作规则

| 状态 | 超时时间 | 自动行为 |
|------|----------|----------|
| ROLL_DICE | 15 秒 | 系统自动掷骰 |
| 城池购买决策 | 20 秒 | 放弃购买 |
| 锦囊卡选择 | 20 秒 | 跳过使用 |
| 城池升级 | 15 秒 | 跳过升级 |
| 弃牌选择 | 30 秒 | 自动弃最旧的卡 |
| 出狱操作 | 20 秒 | 自动等待（不出狱） |
| 武将选择阶段 | 60 秒 | 随机分配默认武将 |

## 二、格子落地子流程

### 2.1 城池格落地流程

```
落地城池格
    │
    ├─ [无主城池]
    │       │
    │       ├─ 玩家资金 ≥ 购买价？
    │       │       ├─ YES → 弹出「购买确认」弹窗（城池名/价格/过路费预览）
    │       │       │         ├─ 确认购买 → 扣款，设置所有权，播放占领动画
    │       │       │         └─ 放弃 / 超时 → 无操作
    │       │       └─ NO → 仅展示「资金不足」提示，无法购买
    │       │
    │       └─ 势力折扣判定：武将与城池势力匹配时自动应用折扣
    │
    ├─ [己方城池]
    │       │
    │       └─ 高亮「升级城池」快捷按钮（若资金充足），无强制操作
    │
    └─ [他人城池]
            │
            ├─ 计算应付过路费（含等级倍率 × 势力加成 × 技能加成）
            ├─ 检查「空城计」卡牌：若持有，弹出「是否使用空城计」
            ├─ 检查刘备技能：自动应用 50% 减免
            ├─ 检查甘宁技能：自动触发劫营（城主少收 50%，玩家获得 50%）
            │
            ├─ 玩家资金 ≥ 应付金额？
            │       ├─ YES → 自动扣款，城主自动收款，播放过路费动画
            │       └─ NO → 强制出售流程（见 §5.1）
            │
            └─ 结算完成 → 继续回合
```

**过路费结算优先级（从高到低）**：
1. 空城计（完全免疫）
2. 刘备技能（50% 减免）
3. 夏侯惇技能（30% 减免，与刘备不叠加，取较高值）
4. 甘宁技能（城主少收 50%，甘宁获得 50%）
5. 张飞技能（城主额外获得 50%）
6. 区域势力加成（荆州 +20%）
7. 最终金额取整（向下取整到整数）

> **叠加规则**：所有加成为乘法叠加，不是加法。顺序：基础过路费 × 等级倍率 × 势力加成 × 城主技能加成，再由城主和访客各自的减免/劫营分别计算。

### 2.2 锦囊格落地流程

```
落地锦囊格
    │
    ├─ 诸葛亮特判：先展示「下一张锦囊卡」预览，询问是否抽取
    │       ├─ 选择抽取 → 进入普通抽卡流程
    │       └─ 跳过 → 回合继续（锦囊牌堆不动）
    │
    └─ 普通抽卡：
            ├─ 从牌堆顶抽 1 张锦囊卡
            ├─ 手牌 < 5 张 → 加入手牌，展示卡牌获得动画
            └─ 手牌 = 5 张 → 弹出弃牌面板（展示 6 张卡，玩家选 1 张弃掉）
                    └─ 30 秒超时 → 自动弃最旧的 1 张
```

**牌堆耗尽处理**：锦囊牌堆抽完后，将弃牌堆洗牌重置为新牌堆。

### 2.3 天命格落地流程

```
落地天命格
    │
    ├─ 从天命牌堆顶翻开 1 张牌
    ├─ 全局广播「XX 触发了天命卡：XXX」
    ├─ 执行卡牌效果（详见 card-design-v1.1.md §三）
    │
    ├─ 黄忠特判（触发前判定）：
    │       ├─ 随机 30% 概率
    │       └─ YES → 若为负面卡，转为对应正面等价效果（见映射表）
    │
    └─ 效果结算完毕 → 卡牌进入弃牌堆，继续回合
```

**黄忠负面→正面映射表**：

| 原负面卡 | 转换正面效果 |
|----------|------------|
| 败走华容（入狱） | 获得 500 金 |
| 大意失荆州（失城） | 获得 1 座无主城池 |
| 乐不思蜀（停 2 回合） | 获得 300 金 |
| 鞠躬尽瘁（-300） | 获得 300 金 |
| 望风而逃（后退 5 格） | 前进 5 格 |
| 天灾人祸（-100） | 获得 100 金 |

### 2.4 关卡格落地流程

```
落地关卡格
    │
    ├─ [5人以上局 或 对应势力武将] → 直接通过，不停留，继续回合
    │
    └─ [其他情况]
            │
            ├─ 设置玩家状态：BLOCKED_AT_CHECKPOINT，剩余停留回合 = 1
            ├─ 展示「停关1回合」提示
            └─ 下一回合该玩家回合开始时，清除状态，正常行动
```

### 2.5 监狱格（格06）进入流程

进入监狱的触发来源：
- 天命卡「败走华容」
- 未来扩展：掷出三次相同点数（保留接口）

```
触发入狱
    │
    ├─ 关羽特判：拥有「过五关斩六将」且未使用 → 弹窗询问是否使用
    │       ├─ 使用 → 标记技能已用，不入狱，继续回合
    │       └─ 不使用 / 无技能 → 继续入狱流程
    │
    ├─ 司马懿特判：同上逻辑
    │
    ├─ 棋子移动到格06（监狱格）
    ├─ 设置玩家状态：IN_JAIL = true
    └─ 展示入狱动画和提示
```

**入狱期间出狱逻辑（回合开始时）**：
```
回合开始，玩家状态 IN_JAIL = true
    │
    ├─ 选项 A：支付 500 金保释（若资金充足，按钮高亮）
    ├─ 选项 B：尝试掷骰（仅 2 颗骰子规则下有效，掷出相同点数即出狱）
    │           └─ 注意：1 颗骰子规则下无法通过此方式出狱
    └─ 选项 C：等待（消耗 1 回合，最多等 3 回合，第 3 回合结束必须缴费出狱）
```

## 三、地图数据结构规格

### 3.1 格子数据模型

```typescript
interface Cell {
  id: number;           // 1-56
  name: string;         // 显示名称
  type: CellType;       // 'city' | 'jinnang' | 'tianming' | 'checkpoint' | 'prison' | 'rest' | 'start'
  region: Region;       // 'wei' | 'jing' | 'shu' | 'wu' | 'neutral'
  faction: Faction;     // 城池归属势力（影响武将折扣）
  buyPrice: number;     // 购买价格（非城池为 0）
  baseToll: number;     // 基础过路费（= buyPrice * 0.1）
  upgradeLevel: number; // 当前等级 1-5（默认 1）
  ownerId: string | null; // 玩家 ID，null = 无主
  position: {           // 地图渲染坐标
    x: number;
    y: number;
  };
  connectedTo?: number[]; // 关卡格：连接的区域（用于判断势力免停）
}
```

### 3.2 格子顺序（顺时针编号）

地图为单向顺时针循环，起点为格09（洛阳），编号规则：
- 格 01-14：北方区域（魏）
- 格 15-28：荆州区域（中立）
- 格 29-42：蜀地区域（蜀）
- 格 43-56：江东区域（吴）
- 起点标记在格 09（洛阳），玩家经过时触发起点奖励

> **注意**：格09是起点，但起点功能（经过给钱）附加在格09上，不单独设立格子。

## 四、玩家数据模型

```typescript
interface Player {
  id: string;               // 唯一标识
  name: string;             // 昵称（最多 8 汉字）
  avatar: string;           // 头像 URL
  heroId: HeroId;           // 选定武将
  gold: number;             // 当前资金
  position: number;         // 当前所在格子 ID（1-56）
  hand: CardId[];           // 手牌（锦囊卡，最多 5 张）
  ownedCells: number[];     // 拥有的格子 ID 列表
  status: PlayerStatus;     // 见下方枚举
  statusEffects: StatusEffect[]; // 持续效果列表
  heroAbilityUsed: {        // 每局限用技能的使用状态
    [abilityId: string]: boolean;
  };
  stats: {                  // 统计数据（用于结算页面）
    totalTollPaid: number;
    totalTollReceived: number;
    cardsUsed: number;
    lapCount: number;       // 经过起点次数
  };
  isConnected: boolean;     // 网络状态
  isAI: boolean;            // 是否为 AI 托管（断线后）
}

type PlayerStatus =
  | 'ACTIVE'           // 正常
  | 'IN_JAIL'          // 入狱
  | 'JAILED_WAIT_1'    // 入狱等待第1回合

interface StatusEffect {
  type: 'LE_BU_SI_SHU' | 'ALLIANCE' | 'MEIREN_COOLDOWN' | 'FIRE_BURN';
  remainingTurns: number;
  sourcePlayerId: string;
  metadata?: any;  // 联盟时存盟友 ID，等
}
```

## 五、破产处理流程

### 5.1 强制出售流程（无法支付过路费）

```
玩家资金不足，无法支付过路费
    │
    ├─ 计算玩家总资产（资金 + 所有城池出售价值）
    │
    ├─ 总资产 ≥ 应付金额？
    │       ├─ YES → 弹出「强制出售」面板
    │       │         ├─ 展示玩家所有城池（含当前售价 = 累计投入 × 50%）
    │       │         ├─ 玩家选择出售哪些城池（需选到资金达到应付额）
    │       │         ├─ 30 秒超时 → 系统自动出售最高价值城池直到够付
    │       │         └─ 出售完成 → 扣款，继续回合
    │       │
    │       └─ NO → 破产判定
    │               ├─ 支付所有剩余资金给城主（不足部分豁免）
    │               ├─ 出售所有城池（资金归自己再转给城主，但已不够）
    │               │   实际处理：所有城池变为无主，城主本次过路费损失
    │               └─ 玩家状态设为 BANKRUPT
```

### 5.2 破产后处理

1. 城池：全部变为无主（ownerId = null），等级重置为 1 级
2. 手牌：全部弃入锦囊弃牌堆
3. 玩家：退出行动队列，棋子从地图消失，进入观战模式
4. 广播：「XX 已破产！XX 的城池变为无主」
5. 胜负检查：若仅剩 1 名玩家存活 → 进入 SETTLEMENT

> **破产城池等级重置**：破产玩家的城池等级归零（回到1级），是为了给其他玩家重新购买提供合理价值。

## 六、网络与多人同步规格

### 6.1 事件广播机制

游戏内所有操作通过「事件」广播给所有客户端，客户端根据事件更新本地状态：

| 事件类型 | 触发时机 | 广播内容 |
|----------|----------|----------|
| `PLAYER_JOINED` | 玩家进入房间 | 玩家基础信息 |
| `GAME_STARTED` | 游戏开始 | 初始游戏状态快照 |
| `TURN_CHANGED` | 换回合 | 当前行动玩家 ID |
| `DICE_ROLLED` | 骰子结果 | 点数、是否重掷 |
| `PLAYER_MOVED` | 棋子移动 | 玩家 ID、目标格子、路径 |
| `CELL_PURCHASED` | 购买城池 | 玩家 ID、格子 ID、花费 |
| `TOLL_PAID` | 支付过路费 | 付款方、收款方、金额 |
| `CARD_DRAWN` | 抽卡 | 玩家 ID、卡牌 ID（仅本人可见详情） |
| `CARD_USED` | 使用锦囊卡 | 玩家 ID、卡牌 ID、目标 |
| `FATE_TRIGGERED` | 天命卡触发 | 卡牌 ID、效果描述 |
| `PLAYER_BANKRUPT` | 玩家破产 | 玩家 ID |
| `GAME_ENDED` | 游戏结束 | 最终排名、资产 |

### 6.2 断线重连

- 玩家断线后，服务端保留其状态 **5 分钟**
- 5 分钟内重连：恢复所有游戏状态，继续游戏
- 断线期间轮到该玩家行动：**AI 托管模式**（最简单策略：掷骰、不买城池、不用卡牌）
- 超过 5 分钟未重连：视为自愿放弃，触发破产流程（资产仍保留，但城池释放）
- 重连后可选择：继续参与 或 维持观战

### 6.3 操作合法性验证

服务端需对所有玩家操作做合法性校验，拒绝不合法操作：
- 不是自己的回合时发来操作
- 资金不足时发来购买/升级请求
- 手牌不含该卡时发来使用卡牌请求
- 对已破产玩家发来目标操作
- 超出技能使用次数的操作

## 七、UI 交互规格

### 7.1 游戏主界面布局

```
┌──────────────────────────────────────────────────────┐
│  [玩家1信息栏]  [玩家2]  [玩家3]  [玩家4]  [聊天/表情]  │  ← 顶部玩家栏
├──────────────────────────────────────────────────────┤
│                                                      │
│                   【地图区域】                         │  ← 主视区（占屏幕 65%）
│              56格棋盘 + 棋子动画                       │
│                                                      │
├──────────────────────────────────────────────────────┤
│  [手牌展示区]          [操作按钮区]    [骰子区]          │  ← 底部操作栏
│  最多5张锦囊卡           掷骰 / 升级    骰子动画          │
└──────────────────────────────────────────────────────┘
```

### 7.2 关键交互反馈要求

| 操作 | 反馈要求 |
|------|----------|
| 掷骰子 | 骰子翻转动画（≥0.5s），点数数字弹出特效 |
| 棋子移动 | 逐格移动动画，速度约 0.15s/格 |
| 经过起点 | 「经过起点！获得 2000 金」飘字动画 |
| 购买城池 | 旗帜插入城池动画，颜色变为玩家色 |
| 支付过路费 | 金币从付款方飘向收款方 |
| 使用卡牌 | 卡牌翻转展示，全局可见效果动画 |
| 玩家破产 | 棋子倒下动画，灰色显示 |
| 触发天命卡 | 全屏卡牌展示，配历史典故插图 |

### 7.3 非当前玩家的操作限制

非当前行动玩家在他人回合期间：
- **可以**：查看地图、查看自己城池信息、查看手牌
- **不可以**：点击掷骰、购买、升级、使用卡牌（按钮置灰）
- **例外**：被「美人计」「借刀杀人」等卡牌指定为目标时，会弹出响应弹窗

## 八、起点逻辑详述

### 8.1 起点奖励触发条件

- **经过**起点（移动路径中途经过格09）：触发奖励
- **停在**起点（恰好落在格09）：同样触发奖励
- **传送到**起点（使用「走为上计」）：触发奖励
- **后退经过**起点（「望风而逃」后退 5 格经过起点）：**不触发**奖励

### 8.2 不同人数起点奖励金额

| 人数 | 基础奖励 | 实际到手 |
|------|----------|----------|
| 2-4人 | 2000 | 2000（+ 曹操加成 300） |
| 5-6人 | 2500 | 2500 |
| 7-8人 | 3000 | 3000 |
